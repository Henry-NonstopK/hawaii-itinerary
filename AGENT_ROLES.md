# 에이전트 역할 및 보고 규칙

## 고정 역할명

- `PL`: 전체 리더, 작업 분배, 최종 보고, 일정/배포 상태 판단
- `개발1`: UI/화면/모바일 사용성 담당
- `개발2`: 데이터/일정/지도 링크/문구 담당
- `검증`: 링크, 일정 누락, 모바일 표시, 공개 URL 확인 담당
- `배포`: GitHub push, GitHub Pages, 공개 URL 상태 담당
- `프로젝트설명`: 변경히스토리, README, 작업 기록 담당

## 보고 규칙

- 사용자가 작업을 지시하면 각 역할은 자기 상태를 보고한다.
- 긴 작업은 최소 1분마다 진행상황을 공유한다.
- 채널 보고는 짧게 유지하고, 아래 형식을 기본으로 쓴다.

```text
[PL 보고]
현재 상태:
- 개발1:
- 개발2:
- 검증:
- 배포:
- 프로젝트설명:

다음 액션:
막힘 여부:
```
